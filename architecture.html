Technical Architecture

edgeX Agent is an AI-powered trading agent running inside Telegram. It connects to the edgeX perpetual contract exchange, processes real-time news via MCP, and executes trades through natural language — all from a single long-polling bot process.

Python 3.11 python-telegram-bot 22.5 SQLite edgeX Python SDK Fernet AES-128

// security

Security Model

Field-level encryption, trade confirmation, safety limits, and key auto-deletion from chat.

🔐

Fernet Encryption at Rest

stark_private_key and ai_api_key encrypted with AES-128-CBC via Fernet. Key derived from SHA-256 of TELEGRAM_BOT_TOKEN. All 3 users migrated, verified enc: prefix.

🗑️

Key Auto-Deletion

When user sends their L2 private key, the bot immediately deletes the message from Telegram chat via deleteMessage. Key never persists in chat history.

✅

Mandatory Trade Confirmation

Every trade shows a plan with entry, TP, SL, size, leverage, and reasoning. User must tap "Confirm Execute" — no silent execution. Cancel always available.

🛡️

Safety Limits

Max position: $500. Max leverage: 5x. Loss circuit breaker: 20%. Max 3 concurrent positions. AI enforces these in trade plan generation.

⏱️

Rate Limiting

15 messages per 60 seconds per user. Stale AI request cancellation prevents queue buildup. ConversationHandler timeouts (120s) prevent state leaks.

🔗

Reply Binding

Defaults(do_quote=True) ensures every bot reply is bound to the user's original message via Telegram's reply mechanism. AI responses additionally use explicit reply_to_message_id.

// infrastructure

Deployment

Single Oracle Cloud VM. Two systemd services. No containers, no load balancer, no external database.

🖥️ Server

Oracle Cloud VM (Ubuntu). Single instance. All files in /home/ubuntu/edgex-agent-bot/. Python 3.11 venv.

⚙️ edgex-agent.service

Main bot process. Runs main.py via venv Python. EnvironmentFile=.env. Restart=always. Long-polling (no webhook).

📡 bwenews-mcp.service

BWEnews sidecar. Runs bwenews_mcp.py. Serves MCP on localhost:8788. Polls @BWEnews every 30s via Telethon.

📦 Dependencies

python-telegram-bot[ext]>=21.0, edgex-python-sdk>=0.3.0, httpx, aiohttp, python-dotenv, cryptography (Fernet). Plus edgex-cli (npm global).

🗄️ Storage

SQLite single file edgex_agent.db. No backup automation yet. DB at deployment root (not bot/ subdirectory).

🔧 CLI Tools

edgex-cli at /home/ubuntu/.npm-global/bin/edgex — exchange market data + order placement. droid at /home/ubuntu/.local/bin/droid — Factory AI inference.
